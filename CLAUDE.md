## What This Is

A personal technical notes site built with SolidStart (SolidJS). Content lives in `content/` as Markdown; `docs/` is the generated static site deployed to GitHub Pages at https://tanoshima.github.io/tanoshima/.

## Commands

```bash
pnpm dev        # start dev server
pnpm build      # build static site to .output/public/
bash build.sh   # build and copy to docs/ (for GitHub Pages)
```

## Architecture

- **`content/`** — Markdown source files (`readme.md`, `about_me.md`)
- **`src/routes/`** — SolidStart file-based routes (`index.tsx` = Top, `about.tsx` = AboutMe)
- **`src/components/`** — `Sidebar.tsx` (nav), `MarkdownPage.tsx` (renders HTML + prev/next)
- **`src/styles/main.css`** — GitBook-like layout CSS (no CSS framework)
- **`src/app.tsx`** — Root layout wrapping all routes with sidebar + body shell
- **`docs/`** — GitHub Pages output; regenerated by `build.sh`, do not edit manually
- **`app.config.ts`** — SolidStart config: `preset: "static"`, `baseURL: "/tanoshima"`

## Key Details

- Base path is `/tanoshima/` — all internal links must be relative or use `useHref`
- Markdown is parsed at build time via `marked.parse()` (synchronous, no top-level await)
- Markdown files are imported as raw strings using Vite's `?raw` suffix
- Static prerendering outputs per-route `index.html` files, enabling direct links on GitHub Pages
